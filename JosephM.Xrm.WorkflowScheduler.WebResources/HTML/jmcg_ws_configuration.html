Joseph Mcgregor Workflow Scheduler

This solution adds the capability to configure workflow processes to run on a recurring schedule

Recurring workflows are configured by creating 'Workflow Task' records. The ways which workflows may be configured to recur are explained in the table below

Note where workflows are targeted by the recurrence they are required to be configured with the "Run On Demand" option checked so the Workflow Scheduler component can start instances of it

See this article for more details https://josephmcmac.wordpress.com/2017/12/21/dynamics-365-workflow-scheduler

Workflow Execution Type Details
Target This Workflow Task One instance of the workflow will execute on the schedule. In this case the target type of the target workflow is required to be the 'Workflow Task' record type.

Generally this would be used to run a workflow containing a custom workflow activity. An example could be triggering integration or other business processes too complex or not appropriate to express in a standard configured workflow
Target Per Fetch Result An instance of the workflow will execute on the schedule for each record returned by a fetch query. In this case the target type of the target workflow is required to be the same as the fetch query.

An example is identifying a set of records which are active and have past the expiry date. Those records could be identified through a fetch query and the workflow could be triggered to set each of them to expired
Target Per View Result An instance of the workflow will execute on the schedule for each record returned by a saved system view. In this case the target type of the target workflow is required to be the same as the view.

This is intended to be used in similar scenarios to the Target Per Fetch Result option, but allows the logic for the targets to be configured in a system view rather than requiring the fetchxml to be entered
View Notification Notification emails will be sent when there are records returned by a system view. The notification email will list the records along with hyperlinks.

Notification's can be sent to either a queue recipient, or to each user who owns records in the view results and has an email address populated.

This is intended to be used where the view defines a set of records which require actioning
Monitor Only This type only monitors for instances of a workflow failing and does not run any scheduled process itself.

It is intended to be used for monitoring a workflow which runs on create or change of a record.
